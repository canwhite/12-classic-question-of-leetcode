# 单向链表小白指南

## 🎯 什么是单向链表？

### 基本概念
**单向链表**是由一个个**节点**串联起来的数据结构，每个节点包含**数据**和指向下一个节点的**指针**。

### 通俗理解
```
单向链表就像一列火车：
🚂 → 🚃 → 🚃 → 🚃 → 🚃 → 🔚

每节车厢（节点）：
- 车厢内的货物（数据）
- 连接下一节车厢的挂钩（指针）
- 最后一节车厢没有挂钩（指向nil）

特点：
- 只能从前向后走
- 不能从后向前走
- 要找第5节车厢，必须从第1节开始走
```

### 链表的基本结构
```go
// 链表节点
type ListNode struct {
    Val  int       // 数据：存储的值
    Next *ListNode // 指针：指向下一个节点
}

// 链表
type LinkedList struct {
    Head *ListNode // 头节点：链表的第一个节点
    Size int       // 长度：链表中节点的数量
}
```

## 🔥 单向链表的核心特征

### 1. **单向性（One-way）**
- 只能从头到尾遍历
- 每个节点只知道下一个节点在哪里
- 不能直接访问前一个节点
- 像单行道，只能向前走

### 2. **动态性（Dynamic）**
- 可以随时插入或删除节点
- 不需要预先分配内存
- 大小可以动态变化
- 像乐高积木，随时可以添加或移除

### 3. **非连续性（Non-contiguous）**
- 节点在内存中不连续存储
- 通过指针连接各个节点
- 插入删除不需要移动其他元素
- 像散落在不同地方的宝藏，通过线索连接

### 4. **带头节点（Head Node）**
- 必须知道头节点才能访问整个链表
- 头节点丢失，整个链表就丢失了
- 像串项链的线头，抓住它才能找到所有珠子

## 💡 单向链表的详细案例

### 案例1：创建和遍历链表
```go
func createLinkedList(values []int) *ListNode {
    if len(values) == 0 {
        return nil
    }

    // 创建头节点
    head := &ListNode{Val: values[0]}
    current := head

    // 创建剩余节点
    for i := 1; i < len(values); i++ {
        newNode := &ListNode{Val: values[i]}
        current.Next = newNode  // 当前节点指向新节点
        current = newNode       // 当前指针移动到新节点
    }

    return head
}

func printLinkedList(head *ListNode) {
    current := head
    for current != nil {
        fmt.Printf("%d", current.Val)
        if current.Next != nil {
            fmt.Printf(" → ")
        }
        current = current.Next
    }
    fmt.Println(" → nil")
}

// 执行过程：createLinkedList([1, 2, 3])
// 1. 创建头节点：head=&{Val:1, Next:nil}
// 2. current = head
// 3. i=1: 创建node2(&{Val:2, Next:nil})，head.Next=node2，current=node2
// 4. i=2: 创建node3(&{Val:3, Next:nil})，node2.Next=node3，current=node3
// 5. 返回head
// 结果：1 → 2 → 3 → nil
```

### 案例2：在链表头部插入节点
```go
func insertAtHead(head *ListNode, val int) *ListNode {
    newNode := &ListNode{Val: val, Next: head}
    return newNode
}

// 执行过程：insertAtHead(1→2→3→nil, 0)
// 原始链表：1 → 2 → 3 → nil
// 1. 创建新节点：newNode = &{Val:0, Next:head}
// 2. 新节点指向原头节点：newNode.Next = 1→2→3→nil
// 3. 返回新节点作为新的头节点
// 结果：0 → 1 → 2 → 3 → nil
```

### 案例3：在链表尾部插入节点
```go
func insertAtTail(head *ListNode, val int) *ListNode {
    newNode := &ListNode{Val: val}

    // 空链表情况
    if head == nil {
        return newNode
    }

    // 找到尾节点
    current := head
    for current.Next != nil {
        current = current.Next
    }

    // 将新节点连接到尾节点
    current.Next = newNode
    return head
}

// 执行过程：insertAtTail(1→2→3→nil, 4)
// 原始链表：1 → 2 → 3 → nil
// 1. 创建新节点：newNode = &{Val:4, Next:nil}
// 2. current从头节点开始遍历
// 3. current移动到尾节点（值为3的节点）
// 4. current.Next = newNode（3指向4）
// 结果：1 → 2 → 3 → 4 → nil
```

### 案例4：删除指定值的节点
```go
func deleteNode(head *ListNode, val int) *ListNode {
    // 如果要删除的是头节点
    if head != nil && head.Val == val {
        return head.Next
    }

    current := head
    var prev *ListNode

    // 寻找要删除的节点
    for current != nil && current.Val != val {
        prev = current
        current = current.Next
    }

    // 找到了要删除的节点
    if current != nil {
        prev.Next = current.Next  // 前一个节点指向下一个节点
        // current节点会被垃圾回收
    }

    return head
}

// 执行过程：deleteNode(1→2→3→4→nil, 3)
// 原始链表：1 → 2 → 3 → 4 → nil
// 1. 检查头节点，不是要删除的值
// 2. current=1, prev=nil
// 3. current≠3，prev=1, current=2
// 4. current≠3，prev=2, current=3
// 5. current=3，找到要删除的节点
// 6. prev.Next = current.Next（2指向4）
// 结果：1 → 2 → 4 → nil
```

### 案例5：反转链表（迭代法）
```go
func reverseList(head *ListNode) *ListNode {
    var prev *ListNode
    current := head

    for current != nil {
        next := current.Next   // 保存下一个节点
        current.Next = prev    // 当前节点指向前一个节点
        prev = current         // prev移动到当前节点
        current = next         // current移动到下一个节点
    }

    return prev  // prev就是新的头节点
}

// 执行过程：reverseList(1→2→3→nil)
// 初始：prev=nil, current=1→2→3→nil
//
// 第1轮：
// next = 2→3→nil
// 1.Next = nil (1→nil)
// prev = 1
// current = 2→3→nil
//
// 第2轮：
// next = 3→nil
// 2.Next = 1 (2→1→nil)
// prev = 2→1→nil
// current = 3→nil
//
// 第3轮：
// next = nil
// 3.Next = 2 (3→2→1→nil)
// prev = 3→2→1→nil
// current = nil
//
// 结束：返回prev = 3→2→1→nil
```

### 案例6：查找中间节点（快慢指针）
```go
func middleNode(head *ListNode) *ListNode {
    slow := head  // 慢指针，每次走一步
    fast := head  // 快指针，每次走两步

    for fast != nil && fast.Next != nil {
        slow = slow.Next      // 慢指针走一步
        fast = fast.Next.Next // 快指针走两步
    }

    return slow  // 慢指针指向中间节点
}

// 执行过程：middleNode(1→2→3→4→5→nil)
// 初始：slow=1, fast=1
//
// 第1轮：slow=2, fast=3
// 第2轮：slow=3, fast=5
// 第3轮：slow=4, fast=nil
//
// 结束：返回slow = 3
// 结果：中间节点是3
```

### 案例7：检测链表是否有环
```go
func hasCycle(head *ListNode) bool {
    if head == nil || head.Next == nil {
        return false
    }

    slow := head  // 慢指针，每次走一步
    fast := head  // 快指针，每次走两步

    for fast != nil && fast.Next != nil {
        slow = slow.Next      // 慢指针走一步
        fast = fast.Next.Next // 快指针走两步

        if slow == fast {
            return true  // 快慢指针相遇，有环
        }
    }

    return false  // 快指针到达终点，无环
}

// 执行过程：有环链表 1→2→3→4→2（2到4形成环）
// 初始：slow=1, fast=1
//
// 第1轮：slow=2, fast=3
// 第2轮：slow=3, fast=2
// 第3轮：slow=4, fast=4
//
// 相遇：返回true，有环
```

### 案例8：合并两个有序链表
```go
func mergeTwoLists(l1 *ListNode, l2 *ListNode) *ListNode {
    dummy := &ListNode{}  // 哨兵节点
    current := dummy

    for l1 != nil && l2 != nil {
        if l1.Val <= l2.Val {
            current.Next = l1  // 选择较小的节点
            l1 = l1.Next
        } else {
            current.Next = l2
            l2 = l2.Next
        }
        current = current.Next
    }

    // 连接剩余的节点
    if l1 != nil {
        current.Next = l1
    }
    if l2 != nil {
        current.Next = l2
    }

    return dummy.Next  // 返回真正的头节点
}

// 执行过程：mergeTwoLists(1→3→5→nil, 2→4→6→nil)
// 初始：dummy→nil, current=dummy
//
// 第1轮：1<2，current→1，current=1, l1=3→5→nil
// 第2轮：3>2，current→2，current=2, l2=4→6→nil
// 第3轮：3<4，current→3，current=3, l1=5→nil
// 第4轮：5>4，current→4，current=4, l2=6→nil
// 第5轮：5<6，current→5，current=5, l1=nil
// 第6轮：l1=nil，连接l2：current→6→nil
//
// 结果：1→2→3→4→5→6→nil
```

## 🌳 链表在实际生活中的应用

### 1. **音乐播放列表**
```go
type Song struct {
    Title string
    Artist string
    Duration int
    Next *Song
}

type Playlist struct {
    Current *Song
    Size int
}

func (p *Playlist) AddSong(title, artist string, duration int) {
    newSong := &Song{
        Title: title,
        Artist: artist,
        Duration: duration,
        Next: nil,
    }

    if p.Current == nil {
        p.Current = newSong
        return
    }

    // 找到最后一首歌
    current := p.Current
    for current.Next != nil {
        current = current.Next
    }

    current.Next = newSong
    p.Size++
}

func (p *Playlist) PlayNext() {
    if p.Current != nil {
        fmt.Printf("正在播放: %s - %s\n", p.Current.Title, p.Current.Artist)
        p.Current = p.Current.Next
    }
}
```

### 2. **浏览器历史记录**
```go
type HistoryEntry struct {
    URL string
    Title string
    Timestamp int64
    Next *HistoryEntry
}

type BrowserHistory struct {
    Head *HistoryEntry
    Size int
}

func (bh *BrowserHistory) VisitPage(url, title string) {
    entry := &HistoryEntry{
        URL: url,
        Title: title,
        Timestamp: time.Now().Unix(),
        Next: bh.Head,
    }
    bh.Head = entry
    bh.Size++
}

func (bh *BrowserHistory) GetHistory(limit int) []HistoryEntry {
    var history []HistoryEntry
    current := bh.Head
    count := 0

    for current != nil && count < limit {
        history = append(history, *current)
        current = current.Next
        count++
    }

    return history
}
```

## ⚡ 链表 vs 数组

### 链表的优势
```
✓ 插入删除效率高：O(1)（已知位置）
✓ 动态大小：不需要预先分配空间
✓ 内存利用率高：不需要连续内存
✓ 容易扩展：可以无限添加节点
```

### 数组的优势
```
✓ 随机访问：O(1)
✓ 缓存友好：连续内存，访问速度快
✓ 内存开销小：不需要额外存储指针
✓ 简单直观：容易理解和实现
```

### 选择标准
```
使用链表的场景：
✓ 频繁插入删除
✓ 不知道数据大小
✓ 内存空间不连续
✓ 需要动态扩展

使用数组的场景：
✓ 频繁随机访问
✓ 数据大小固定
✓ 追求访问速度
✓ 内存空间有限
```

### 时间复杂度对比
```
操作      | 链表    | 数组
----------|---------|---------
访问      | O(n)    | O(1)
插入      | O(1)    | O(n)
删除      | O(n)    | O(n)
查找      | O(n)    | O(n)
空间      | O(n)    | O(n)
```

## 🚀 链表的常见问题和解决

### 1. **空指针问题**
```go
// ❌ 错误：没有检查nil
func badPrint(head *ListNode) {
    for head.Val != 0 {  // 如果head是nil，会panic
        fmt.Println(head.Val)
        head = head.Next
    }
}

// ✅ 正确：检查nil
func goodPrint(head *ListNode) {
    current := head
    for current != nil {  // 先检查当前节点是否存在
        fmt.Println(current.Val)
        current = current.Next
    }
}
```

### 2. **断链问题**
```go
// ❌ 错误：丢失后续节点
func badDelete(head *ListNode, target int) {
    if head.Val == target {
        head = head.Next  // 只修改了局部变量，真正的头节点没有改变
    }
}

// ✅ 正确：正确处理头节点
func goodDelete(head *ListNode, target int) *ListNode {
    if head == nil {
        return nil
    }

    // 删除头节点的情况
    if head.Val == target {
        return head.Next
    }

    // 删除非头节点的情况
    current := head
    for current.Next != nil && current.Next.Val != target {
        current = current.Next
    }

    if current.Next != nil {
        current.Next = current.Next.Next
    }

    return head
}
```

### 3. **内存泄漏**
```go
// ❌ 错误：没有正确处理删除
func badClearList(head *ListNode) {
    head = nil  // 只是让局部变量指向nil，节点还在内存中
}

// ✅ 正确：逐个删除节点（在Go中会由垃圾回收处理）
func goodClearList(head *ListNode) {
    // Go有垃圾回收，不需要手动释放
    // 让所有节点的Next指向nil，帮助垃圾回收
    current := head
    for current != nil {
        next := current.Next
        current.Next = nil
        current = next
    }
}
```

## 🎓 链表的记忆技巧

### 节点结构记忆法
```
📦 包裹记忆法：
   包裹里的东西 = 数据（Val）
   包裹上的地址标签 = 指针（Next）

🚂 火车车厢记忆法：
   车厢里的货物 = 数据
   连接下一节车厢的挂钩 = 指针
```

### 操作记忆法
```
插入节点：
   A → C          B插入中间
   A → B → C      A.Next指向B，B.Next指向C

删除节点：
   A → B → C      删除B
   A → C          A.Next指向C

遍历链表：
   从头开始，沿着Next一路向前
   像寻宝一样，跟着线索找下一个宝藏
```

### 指针技巧记忆法
```
哑节点技巧：
   在真实头节点前加一个假头节点
   简化头节点的特殊处理

快慢指针技巧：
   快指针走两步，慢指针走一步
   像兔子乌龟赛跑，用来找中间位置
```
